Python Capability Model

Python · Capability Model · 6 Primitives

This page presents a pure-Python reference implementation of the Church Machine's core security primitives. It models the same concepts enforced in hardware by the CLOOMC ISA — using ordinary Python classes so that anyone familiar with the language can read the capability model without learning assembly first. The three classes (CapabilityToken, ContextRegister, and ChurchMachineSimulator) implement the six fundamental operations: LOAD, SAVE, CALL, RETURN, CHANGE, and SWITCH.

Class 1 of 3

CapabilityToken

A Golden Token is the fundamental unit of authority in the Church Machine. Every access — read, write, or execute — requires presenting a valid token. The token carries the base address, limit (size), and a permission bitmask (READ, WRITE, EXECUTE).

The hardware validates the token on every memory operation; this Python class models that contract. Tokens are immutable value objects — you cannot widen a capability, only derive a narrower one.

base limit READ WRITE EXECUTE check_permission() check_bounds()
Class 2 of 3

ContextRegister

The Church Machine holds its security state in Context Registers (CRs) rather than in ordinary data registers. A ContextRegister stores a capability token and the associated label (e.g. Namespace, Thread, C-List).

LOAD and SAVE operate on context registers. Only the instructions that explicitly name a register can read or update it — there is no general-purpose pointer arithmetic that can reach the CR file.

load() save() label capability
Class 3 of 3

ChurchMachineSimulator

The top-level simulator wires the registers and tokens together and exposes the six primitive operations as Python methods. Each method enforces the same permission and bounds checks that the CLOOMC ISA hardware pipeline performs.

"""
church_machine_cap.py
Pure-Python reference model of the Church Machine capability primitives.

Classes
-------
CapabilityToken      — immutable Golden Token (base, limit, permissions)
ContextRegister      — named capability holder (CR file slot)
ChurchMachineSimulator — six-primitive capability engine

Primitives: LOAD · SAVE · CALL · RETURN · CHANGE · SWITCH
"""

from __future__ import annotations
import dataclasses
from dataclasses import dataclass, field
from enum import IntFlag, auto
from typing import Optional


# ─────────────────────────────────────────────────────────────────────────────
# Permission flags  (match the CLOOMC ISA permission bit positions)
# ─────────────────────────────────────────────────────────────────────────────
class Perm(IntFlag):
    """Capability permission bits."""
    NONE    = 0
    READ    = auto()   # R — data readable within bounds
    WRITE   = auto()   # W — data writable within bounds
    EXECUTE = auto()   # X — code executable (CALL / RETURN)
    RX      = READ | EXECUTE
    RW      = READ | WRITE
    RWX     = READ | WRITE | EXECUTE


# ─────────────────────────────────────────────────────────────────────────────
# CapabilityToken  — the Golden Token
# ─────────────────────────────────────────────────────────────────────────────
@dataclass(frozen=True)
class CapabilityToken:
    """
    Immutable Golden Token.

    Attributes
    ----------
    base  : int   Starting word address of the protected region.
    limit : int   Size of the region in words (exclusive upper bound = base+limit).
    perms : Perm  Permission flags granted by this token.
    label : str   Human-readable name (namespace label, not enforced).
    """
    base  : int
    limit : int
    perms : Perm
    label : str = ""

    # -- permission checks --------------------------------------------------
    def check_permission(self, required: Perm) -> bool:
        """Return True if *all* required permission bits are present."""
        return (required & self.perms) == required

    def check_bounds(self, offset: int, length: int = 1) -> bool:
        """Return True if [offset, offset+length) lies entirely within [0, limit)."""
        return 0 <= offset and (offset + length) <= self.limit

    def validate(self, required: Perm, offset: int, length: int = 1) -> None:
        """Raise CapabilityError if permission or bounds check fails."""
        if not self.check_permission(required):
            raise CapabilityError(
                f"Permission denied: token '{self.label}' has {self.perms}, "
                f"but {required} required."
            )
        if not self.check_bounds(offset, length):
            raise CapabilityError(
                f"Bounds violation: token '{self.label}' limit={self.limit}, "
                f"access offset={offset} length={length}."
            )

    def derive(self, *, base_offset: int = 0, new_limit: Optional[int] = None,
               new_perms: Optional[Perm] = None, label: str = "") -> CapabilityToken:
        """
        Derive a narrower capability from this one.
        Cannot widen base, limit, or permissions beyond the parent.
        """
        new_base  = self.base + base_offset
        new_lim   = new_limit  if new_limit  is not None else self.limit - base_offset
        new_p     = new_perms  if new_perms  is not None else self.perms
        if new_base < self.base or (new_base + new_lim) > (self.base + self.limit):
            raise CapabilityError("Cannot derive a capability wider than its parent.")
        if (new_p & ~self.perms) != Perm.NONE:
            raise CapabilityError("Cannot grant permissions not held by parent.")
        return CapabilityToken(new_base, new_lim, new_p, label or self.label)


class CapabilityError(Exception):
    """Raised when a capability check fails (permission or bounds)."""


# ─────────────────────────────────────────────────────────────────────────────
# ContextRegister  — a named slot in the CR file
# ─────────────────────────────────────────────────────────────────────────────
class ContextRegister:
    """
    A named capability holder that models a Church Machine context register.

    In hardware each CR is a 32-bit Golden Token plus type metadata.
    Here we store the Python CapabilityToken directly.

    Parameters
    ----------
    name : str   Register name, e.g. 'CR6 (C-List)', 'CR15 (Namespace)'.
    """

    def __init__(self, name: str) -> None:
        self.name: str = name
        self._cap: Optional[CapabilityToken] = None

    # -- LOAD ------------------------------------------------------------------
    def load(self, token: CapabilityToken) -> None:
        """
        LOAD: Install *token* into this register.

        The ISA's ELOAD instruction copies a capability from the C-List into
        a context register; this method models that transfer.
        """
        if not isinstance(token, CapabilityToken):
            raise TypeError(f"Expected CapabilityToken, got {type(token)}")
        self._cap = token

    # -- SAVE ------------------------------------------------------------------
    def save(self) -> CapabilityToken:
        """
        SAVE: Read the capability currently held in this register.

        The ISA's ESAVE instruction copies from a context register back into
        the C-List; this method models the source side of that transfer.
        """
        if self._cap is None:
            raise CapabilityError(f"{self.name} is empty — cannot SAVE.")
        return self._cap

    @property
    def capability(self) -> Optional[CapabilityToken]:
        """The token currently held in this register (None if empty)."""
        return self._cap

    def __repr__(self) -> str:
        cap = self._cap
        if cap is None:
            return f"{self.name}=NULL"
        return (
            f"{self.name}=[base={cap.base:#06x} limit={cap.limit} "
            f"perms={cap.perms.name} '{cap.label}']"
        )


# ─────────────────────────────────────────────────────────────────────────────
# ChurchMachineSimulator  — six primitive operations
# ─────────────────────────────────────────────────────────────────────────────
class ChurchMachineSimulator:
    """
    Minimal Church Machine simulator exposing the six capability primitives.

    Context registers
    -----------------
    CR6  — C-List (capability list for the current abstraction)
    CR7  — Nucleus / kernel entry point
    CR8  — Thread identity
    CR14 — current code lump (Boot.Abstr)
    CR15 — Namespace root

    The six primitives
    ------------------
    LOAD   — copy a capability from the C-List into a named CR
    SAVE   — copy a capability from a named CR back to the C-List
    CALL   — transfer control to a capability-gated abstraction
    RETURN — unwind from an abstraction, restoring saved CRs
    CHANGE — replace the active C-List (namespace scope change)
    SWITCH — swap thread identity (context switch)
    """

    def __init__(self) -> None:
        # Context register file
        self.CR6  = ContextRegister("CR6 (C-List)")
        self.CR7  = ContextRegister("CR7 (Nucleus)")
        self.CR8  = ContextRegister("CR8 (Thread)")
        self.CR14 = ContextRegister("CR14 (Code)")
        self.CR15 = ContextRegister("CR15 (Namespace)")

        # Call stack — each frame saves (return_nia, CR6, CR7)
        self._stack: list[tuple] = []
        # Next instruction address (word offset within the current code lump)
        self.NIA: int = 0

        # Simple word-addressed memory (populated by tests / boot image)
        self.memory: dict[int, int] = {}

    # ── LOAD ──────────────────────────────────────────────────────────────
    def LOAD(self, token: CapabilityToken, target_cr: ContextRegister) -> None:
        """
        LOAD a capability into a context register.

        Models ELOAD in the CLOOMC ISA: the caller presents a token from its
        C-List; the hardware validates it and installs it in the named CR.
        READ permission is required on the source capability.
        """
        token.validate(Perm.READ, offset=0)
        target_cr.load(token)

    # ── SAVE ──────────────────────────────────────────────────────────────
    def SAVE(self, source_cr: ContextRegister, token: CapabilityToken) -> CapabilityToken:
        """
        SAVE the capability from a context register.

        Models ESAVE in the CLOOMC ISA: the capability is read from the CR and
        written back to a C-List slot. WRITE permission is required on the
        destination capability (the C-List slot being updated).
        Returns the saved CapabilityToken.
        """
        token.validate(Perm.WRITE, offset=0)
        saved = source_cr.save()
        return saved

    # ── CALL ──────────────────────────────────────────────────────────────
    def CALL(self, callee: CapabilityToken, method_offset: int = 0) -> None:
        """
        CALL an abstraction via a capability.

        Models ELOADCALL / ECALL in the CLOOMC ISA:
        1. Validates EXECUTE permission and bounds on the callee token.
        2. Pushes a stack frame: (return_NIA, CR6, CR7).
        3. Installs the callee's C-List into CR6.
        4. Sets NIA to callee.base + method_offset.
        """
        callee.validate(Perm.EXECUTE, offset=method_offset)
        # Save caller's frame
        self._stack.append((self.NIA, self.CR6._cap, self.CR7._cap))
        # Install callee context
        self.CR6.load(callee)
        self.NIA = callee.base + method_offset

    # ── RETURN ────────────────────────────────────────────────────────────
    def RETURN(self) -> None:
        """
        RETURN from the current abstraction.

        Models RETURN in the CLOOMC ISA:
        1. Pops the top stack frame.
        2. Restores CR6, CR7, and NIA to their caller values.
        Raises CapabilityError if the stack is empty (return without CALL).
        """
        if not self._stack:
            raise CapabilityError("RETURN with empty call stack — fault.")
        ret_nia, saved_cr6, saved_cr7 = self._stack.pop()
        self.NIA = ret_nia
        if saved_cr6 is not None: self.CR6.load(saved_cr6)
        if saved_cr7 is not None: self.CR7.load(saved_cr7)

    # ── CHANGE ────────────────────────────────────────────────────────────
    def CHANGE(self, new_clist: CapabilityToken) -> None:
        """
        CHANGE the active C-List (namespace scope change).

        Models CHANGE in the CLOOMC ISA: replaces CR6 with a new C-List
        capability. This is used when an abstraction transfers control to a
        sibling scope without a full CALL/RETURN round-trip.
        READ permission is required on the new C-List token.
        """
        new_clist.validate(Perm.READ, offset=0)
        self.CR6.load(new_clist)

    # ── SWITCH ────────────────────────────────────────────────────────────
    def SWITCH(self, new_thread: CapabilityToken) -> None:
        """
        SWITCH thread identity (context switch).

        Models SWITCH in the CLOOMC ISA: replaces CR8 (Thread) with a
        capability referencing the new thread's identity lump.
        READ permission is required on the incoming thread token.
        """
        new_thread.validate(Perm.READ, offset=0)
        self.CR8.load(new_thread)

    # ── helpers ───────────────────────────────────────────────────────────
    def state(self) -> str:
        """Return a one-line summary of current machine state."""
        return (
            f"NIA={self.NIA:#06x}  stack_depth={len(self._stack)}  "
            f"CR6={self.CR6}  CR8={self.CR8}  CR15={self.CR15}"
        )


# ─────────────────────────────────────────────────────────────────────────────
# Quick demo — run with:  python church_machine_cap.py
# ─────────────────────────────────────────────────────────────────────────────
if __name__ == "__main__":
    cm = ChurchMachineSimulator()

    # Mint some Golden Tokens
    ns_token   = CapabilityToken(base=0x0000, limit=64,  perms=Perm.RW,  label="Namespace")
    code_token = CapabilityToken(base=0x0100, limit=256, perms=Perm.RX,  label="Boot.Abstr")
    data_token = CapabilityToken(base=0x0200, limit=128, perms=Perm.RW,  label="DataRegion")
    thrd_token = CapabilityToken(base=0x0300, limit=32,  perms=Perm.READ, label="Thread.Kenneth")

    # LOAD — install tokens into context registers
    cm.LOAD(ns_token,   cm.CR15)
    cm.LOAD(code_token, cm.CR14)
    cm.LOAD(data_token, cm.CR6)
    print("After LOAD:", cm.state())

    # SAVE — read back the namespace token
    write_slot = CapabilityToken(base=0x0010, limit=4, perms=Perm.RW, label="SaveSlot")
    saved = cm.SAVE(cm.CR15, write_slot)
    print(f"SAVE returned: {saved.label} base={saved.base:#06x}")

    # CALL — enter Boot.Abstr at method offset 0
    cm.CALL(code_token, method_offset=0)
    print(f"After CALL: NIA={cm.NIA:#06x} stack_depth={len(cm._stack)}")

    # RETURN — unwind back to caller
    cm.RETURN()
    print(f"After RETURN: NIA={cm.NIA:#06x} stack_depth={len(cm._stack)}")

    # CHANGE — swap to a sibling C-List
    sibling_clist = CapabilityToken(base=0x0400, limit=64, perms=Perm.RW, label="Sibling.CList")
    cm.CHANGE(sibling_clist)
    print(f"After CHANGE: CR6={cm.CR6}")

    # SWITCH — install thread identity
    cm.SWITCH(thrd_token)
    print(f"After SWITCH: CR8={cm.CR8}")
