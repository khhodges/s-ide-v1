Boot Microcode Sequence (Patent Section 15)

Six-Phase Hardware Boot (M-Elevation Active)

0--- IDLE — waiting for boot_start signal
1REG FAULT_RST — Clear all CRs to NULL, all DRs to zero. M-Elevation ON.
2MEM RD LOAD_NS — CR15 ← mLoad(Slot 0). Hardwired GT, zero perms, Inform type.
3MEM RD INIT_THRD — CR8 ← mLoad(Slot 1) Thread identity.
4MEM RD INIT_CLIST — CR6 ← mLoad(Slot 2) C-List. CR5 stays NULL.
5MEM RD LOAD_NUC — CALL into Slot 2: E-GT → mLoad → CR6; read CR6+0 → X-GT (Slot 3, Boot.CLOOMC) → mLoad → CR7; both Inform, not F; PC ← 0.
6--- COMPLETE — M-Elevation OFF → begin instruction fetch → READY
Phase 1 clears all CRs to NULL. Phases 2–5 load valid GTs through mLoad (M-elevation bypasses permission but not bounds/version/seal). Thread loads before C-List. LOAD_NUC follows CALL semantics: Phase 0 loads E-GT into CR6, Phase 1 reads CR6+0 X-GT into CR7, sets PC=0. Both GTs must be Inform type and not F (Far). CR5 remains NULL — the secure default.
Instruction Fetch & Condition Check

Common Prefix (Every Instruction)

1MEM RD Fetch instruction word at memory[PC]
2--- If word = 0 → PP250: return to boot sequence (no HALT)
3--- Decode: opcode[5] | cond[4] | dst[4] | src[4] | imm[15]
4REG Evaluate ARM condition against flags (N,Z,C,V)
5--- If condition false → skip, PC++ (conditional execution)
6--- Dispatch to instruction handler by opcode
7REG Post-execute: DR0 ← 0 (always cleared), emit state change
Security Gates — Trusted Security Base (TSB)

mLoad — Read Gate (Patent Claim 9a)

1REG Read GT from CR: parse Version(7) | Index(17) | Perms(6) | Type(2)
2REG Permission check: GT perms has required bit (unless M-elevated) → PERMISSION fault
3--- Bounds check: index < nsCount → BOUNDS fault
4MEM RD Fetch NS entry (3 words: location, B|F|G|limit, version|seal)
5--- Version match: GT.ver == NS.ver → VERSION fault
6--- MAC/Seal validation: FNV-1a(location, limit) == stored seal → SEAL fault
7MEM WR G-bit reset — mark entry LIVE in NS word1
8REG CR write — write validated capability to destination register
9MEM WR Thread shadow update — save loaded GT to Thread state (CR8 offset)
Pipeline: Permission → Bounds → Version → MAC/Seal → G-bit Reset → CR Write → Thread Shadow. Permission checked FIRST for fail-fast security.

mSave — Write Gate (Patent Claim 9b)

1REG Read source GT from CR: parse Version(7) | Index(17) | Perms(6) | Type(2)
2--- Version match: source GT.ver == NS.ver → VERSION fault
3--- Seal validation: FNV-1a(location, limit) == stored seal → SEAL fault
4MEM RD Target C-List bounds check: target index within limit → BOUNDS fault
5REG B-bit check from CR.word2: B=1 required (unless M-elevated) → BIND fault
6MEM RD F-bit detection: read target NS entry, F=1 → FAR/foreign object → FAR fault if local-only
7--- Seal recompute: recalculate FNV-1a for target entry integrity
8MEM WR G-bit reset — mark source entry LIVE in NS word1
9MEM WR Commit write — write source GT to target C-List slot in memory
Pipeline: Version → Seal → Target Bounds → B-bit → F-bit → Seal Recompute → G-bit Reset → Commit. Symmetric counterpart to mLoad.
Called by: SAVE instruction. Together mLoad + mSave form the complete dual-gate TSB (~329 lines HDL).
Church Domain — 10 Instructions

0 — LOAD CRd, CRs, imm

1REG CR[src].word0 = c-list GT. Null? → NULL_CAP fault
2mLoad(clistGT, 'L', crSrc) — MEM RD NS entry + MEM WR G-bit
3--- Bounds check: imm < nsCount, entry valid → BOUNDS fault
4MEM RD Read target NS entry (3 words) → FNV seal check → SEAL fault
5MEM RD Read GT word from entry's memory location
6REG Write CR[dst] ← GT + location + limit + seals → CR_OCCUPIED fault
7--- PC++
Bus: 3 MEM RD (mLoad NS + target NS + GT word) + 1 MEM WR (G-bit) + 2 REG

1 — SAVE CRd, CRs, imm

1REG CR[dst].word0 = source GT. Null? → NULL_CAP fault
2mSave(srcGT, targetIdx, crDst) — MEM RD src NS + tgt NS, REG B check
3REG CR[src].word0 = c-list GT. Null? → NULL_CAP fault
4mLoad(clistGT, 'S', crSrc) — MEM RD NS entry + MEM WR G-bit
5MEM WR If target slot invalid → create dynamic NS entry (3 words)
6--- Register target in nsClistMap (GC reachability — in-memory map)
7MEM RD Read target slot location → MEM WR Write source GT to memory
8--- PC++
Bus: 4 MEM RD (mSave src+tgt + mLoad NS + slot loc) + 3 MEM WR (G-bit + dyn entry + GT) + 2 REG

2 — CALL CRd (Claims 7, 10)

1REG CR[src].word0 = source GT (E perm, pointing to C-List). Null? → NULL_CAP fault
2--- Validate: GT must be Inform type and not F (Far) → FAULT if either fails
3Phase 0: mLoad(sourceGT, 'E', crSrc) — MEM RD NS entry + MEM WR G-bit → REG CR6 ← validated C-List capability
4MEM RD Read CR6+0 (memory at CR6's location + offset 0) → X-permission GT
5--- Validate: X-GT must be Inform type and not F (Far) → FAULT if either fails
6Phase 1: mLoad(xGT, 'X') — MEM RD NS entry + MEM WR G-bit → REG CR7 ← validated code capability
7--- If handler registered (e.g., GC) → dispatch to safe Turing abstraction
8REG Push call frame (tag=0): save all CRs (16×4 words), all DRs (16), flags, returnPC
9REG If LAMBDA-active: save LAMBDA_PC + LAMBDA-active flag into call frame, clear LAMBDA-active (Claim 7)
10REG Auto-clear B on all preserved CRs: CR[0..15].word2 bit 31 ← 0 (Claim 10)
11--- PC ← 0 (instruction address set to zero — execution begins at first instruction)
Two-phase mLoad: Phase 0 loads E-GT → CR6 (C-List), Phase 1 reads CR6+0 X-GT → CR7 (code entry). NIA = 0.
Bus: 2 MEM RD (2× NS entry) + 1 MEM RD (CR6+0) + 2 MEM WR (2× G-bit) + 5 REG (GT + CR6 + CR7 + frame + B-clear). Call stack is machine-internal.

3 — RETURN (Claims 5, 6, 7)

1REG Check LAMBDA-active flag in machine status (Claim 5 — fast path)
2a--- If LAMBDA-active: PC ← LAMBDA_PC (machine status register), clear LAMBDA-active flag. Zero stack access.
2b--- If not LAMBDA-active: Call stack empty? → PP250: return to boot sequence (no HALT)
3REG Pop frame, inspect 1-bit tag (Claim 6 — self-describing frames)
4aREG Tag=0 (CALL frame): Full domain restore — all CRs, DRs, flags. If saved LAMBDA state present, restore LAMBDA_PC + LAMBDA-active (Claim 7)
4bREG Tag=1 (LAMBDA frame): Simple PC restore from saved LAMBDA_PC
5--- PC ← returnPC
Fast path (LAMBDA-active): 0 MEM, 1 REG (clear flag + restore PC from machine status). Zero stack access.
Slow path (CALL frame): 0 MEM, 1 REG (full frame restore). Call stack is machine-internal.

4 — CHANGE CRd, CRs, imm

1REG CR[src].word0 = source GT. Null? → NULL_CAP fault
2--- Bounds check: imm < nsCount → BOUNDS fault
3MEM RD Read NS entry at index imm (3 words) → BOUNDS fault if null
4MEM RD Read GT word from entry location → REG Write CR[dst] → CR_OCCUPIED fault
5--- PC++ (re-aims register without mLoad — no permission check)
Bus: 2 MEM RD (NS entry + GT word) + 0 MEM WR + 2 REG. No mLoad gate — index-based shortcut.

5 — SWITCH CRs, imm

1REG CR[src] must be non-null → NULL_CAP fault
2REG Swap CR[src] ↔ CR[imm&7] (atomic register swap, 4 words each)
3--- PC++
Bus: 0 MEM. 2 REG (read + swap). Pure register operation.

6 — TPERM CRd, preset[+B]

1--- Decode: presetCode = imm & 0xF, bSet = imm bit 4
2--- Reserved preset? → TPERM_RSV fault
3REG Read CR[dst].word0 GT. Null? → Z=0, N=1 (fail flags), PC++
4REG Test: GT perms from CR.word0 has all required permission bits?
5MEM WR If B modifier + test passed → set B=1 in NS entry word1 (memory)
6REG If B modifier + test passed → set B=1 in CR[dst].word2 (register)
7REG Set flags: Z=pass, N=!pass, C=0, V=0
8--- PC++
Bus: 0 MEM RD + 1 MEM WR (B to NS, only if +B passes) + 4 REG. Perm test is pure register.
Presets: CLEAR(0) R(1) RW(2) X(3) RX(4) RWX(5) L(6) S(7) E(8) LS(9) LE(10) SE(11) LSE(12) RWXLSE(13) + B flag(bit4)

7 — LAMBDA CRd (Claims 3, 5, 7)

1REG CR[dst].word0 = target GT. Null? → NULL_CAP fault
2--- If LAMBDA-active already set → FAULT (non-nestable) (Claim 7)
3mLoad(targetGT, 'X', crDst) — MEM RD NS entry + MEM WR G-bit
4REG Set LAMBDA-active flag in machine status (Claim 3c)
5REG Store return address (PC+1) in LAMBDA_PC machine status register (Claim 3d)
6--- PC ← target code entry point (in-scope, no context save)
Bus: 1 MEM RD (NS entry) + 1 MEM WR (G-bit) + 2 REG (flag + LAMBDA_PC). Machine-status fast path — zero stack access.
RETURN uses fast path: if LAMBDA-active, restore PC from LAMBDA_PC with zero stack access (Claim 5). Nesting only via CALL (Claim 7).

8 — ELOADCALL CRd, CRs, imm (Fused)

1REG CR[src].word0 = c-list GT. Null? → NULL_CAP fault
2mLoad(clistGT, 'L', crSrc) — MEM RD NS entry + MEM WR G-bit
3MEM RD Read target NS entry (3 words) → bounds + seal check
4MEM RD Read GT word from entry location → REG Write CR[dst]
5mLoad(loadedGT, 'E', crDst) — MEM RD NS entry + MEM WR G-bit
6REG Push call frame (all CRs, DRs, flags, returnPC)
7REG Auto-clear B on all active CRs: word2 bit 31 ← 0
8--- PC++ (3 ops in 1 cycle: LOAD + TPERM(E) + CALL)
Bus: 4 MEM RD (2× mLoad NS + target NS + GT word) + 2 MEM WR (2× G-bit) + 4 REG

9 — XLOADLAMBDA CRd, CRs, imm (Fused)

1REG CR[src].word0 = c-list GT. Null? → NULL_CAP fault
2mLoad(clistGT, 'L', crSrc) — MEM RD NS entry + MEM WR G-bit
3MEM RD Read target NS entry (3 words) → bounds + seal check
4MEM RD Read GT word from entry location → REG Write CR[dst]
5REG Check X permission from loaded GT (CR[dst].word0) → PERMISSION fault
6--- In-scope application (no context save)
7--- PC++ (3 ops in 1 cycle: LOAD + TPERM(X) + LAMBDA)
Bus: 3 MEM RD (mLoad NS + target NS + GT word) + 1 MEM WR (G-bit) + 3 REG
Turing Domain — 10 Instructions (inside safe abstractions)

10 — DREAD DRd, CRs, imm

1REG CR[src].word0 = data GT. Null? → NULL_CAP fault
2mLoad(dataGT, 'R', crSrc) — MEM RD NS entry + MEM WR G-bit
3REG Bounds check: imm ≤ CR[src].word2 limit (from register) → BOUNDS fault
4MEM RD DR[dst] ← memory[CR[src].word1 + imm] (location from register)
5--- PC++
Bus: 2 MEM RD (NS entry + data word) + 1 MEM WR (G-bit) + 2 REG. Location & limit from CR registers.

11 — DWRITE DRd, CRs, imm

1REG CR[src].word0 = data GT. Null? → NULL_CAP fault
2mLoad(dataGT, 'W', crSrc) — MEM RD NS entry + MEM WR G-bit
3REG Bounds check: imm ≤ CR[src].word2 limit (from register) → BOUNDS fault
4REG Read DR[dst] value
5MEM WR memory[CR[src].word1 + imm] ← DR value (location from register)
6--- PC++
Bus: 1 MEM RD (NS entry) + 2 MEM WR (G-bit + data word) + 3 REG. Location & limit from CR registers.

12 — BFEXT DRd, CRs, imm (Bitfield Extract)

1REG CR[src].word0 = data GT. Null? → NULL_CAP fault
2mLoad(dataGT, 'R', crSrc) — MEM RD NS entry + MEM WR G-bit
3REG Location from CR[src].word1. Decode pos[5] and width[5] from imm
4--- Validate: width > 0, pos + width ≤ 32 → BOUNDS fault
5MEM RD Read word from memory[location]
6REG DR[dst] ← (word >>> pos) & mask
7--- PC++
Bus: 2 MEM RD (NS entry + data word) + 1 MEM WR (G-bit) + 3 REG

13 — BFINS DRd, CRs, imm (Bitfield Insert)

1REG CR[src].word0 = data GT. Null? → NULL_CAP fault
2mLoad(dataGT, 'W', crSrc) — MEM RD NS entry + MEM WR G-bit
3REG Location from CR[src].word1. Decode pos[5] and width[5] from imm
4--- Validate: width > 0, pos + width ≤ 32 → BOUNDS fault
5REG Read insert value from DR[dst]
6MEM RD Read old word from memory[location]
7MEM WR Write new word = (old & ~mask) | (insert << pos) to memory
8--- PC++
Bus: 2 MEM RD (NS entry + old word) + 2 MEM WR (G-bit + new word) + 3 REG

14 — MCMP DRd, DRs, imm (Multi-word Compare)

1REG Read DR[dst] and DR[src] values
2--- Compare: subtract, set condition flags
3REG Set N, Z, C, V flags
4--- PC++
Bus: 0 MEM. 2 REG (read DRs + write flags). Pure register operation.

15 — IADD DRd, DRs, imm

1REG Read DR[src] + sign-extend imm15
2--- Compute: result = DR[src] + imm (with overflow detection)
3REG DR[dst] ← result, set N, Z, C, V flags
4--- PC++
Bus: 0 MEM. 2 REG (read DR + write DR/flags). Pure register ALU operation.

16 — ISUB DRd, DRs, imm

1REG Read DR[src] + sign-extend imm15
2--- Compute: result = DR[src] - imm (with borrow detection)
3REG DR[dst] ← result, set N, Z, C, V flags
4--- PC++
Bus: 0 MEM. 2 REG. Pure register ALU operation.

17 — BRANCH imm (Conditional)

1--- Condition already evaluated in common prefix
2--- PC ← PC + sign-extend(imm15)
Bus: 0 MEM. 0 REG. Pure PC manipulation (PC is implicit, not a register file access).

18 — SHL DRd, DRs, imm

1REG Read DR[src]
2--- Compute: result = DR[src] << (imm & 31)
3REG DR[dst] ← result, set N, Z flags
4--- PC++
Bus: 0 MEM. 2 REG. Pure register shift.

19 — SHR DRd, DRs, imm

1REG Read DR[src]
2--- Compute: result = DR[src] >>> (imm & 31)
3REG DR[dst] ← result, set N, Z flags
4--- PC++
Bus: 0 MEM. 2 REG. Pure register shift.

Same instruction in both domains. See Church Domain above.
PP250 Garbage Collection — SCIF Cycle

Deterministic GC (Safe Turing Abstraction)

Entered via CALL with valid GT (E perm). Atomic — hidden behind Church interface.
SMEM RD Scan — walk all NS entries, read G-bits (word1 bit 29). REG Check CRs for live references
I--- Identify — entries where G-bit ≠ polarity are unreachable (dead)
CMEM WR Clear — zero dead entries (3 words each) + zero backing memory. Increment version in word2.
F--- Flip — toggle GC polarity (0↔1) for next cycle
Bus: N MEM RD (scan all entries) + M MEM WR (clear dead entries + backing). Bidirectional G-bit.
Exited via RETURN to caller. GC is deterministic — no stop-the-world, no compaction.
Fault Catalog

All Faults Route to Single Handler

NULL_CAP — CR holds GT = 0 (null capability)
BOUNDS — namespace index out of range or offset > limit
VERSION — GT version ≠ namespace entry version
SEAL — FNV seal validation failure
PERMISSION — GT lacks required permission bit
BIND — B=0 on source GT in CR (mSave reads from register)
FAR — F=1 on target (foreign/remote object, needs HTTP/tunnel)
CR_OCCUPIED — writing to non-empty CR
TPERM_RSV — reserved TPERM preset code
INVALID_OP — unknown opcode
All faults route to a single FAULT handler. No silent failures, no fallbacks.
